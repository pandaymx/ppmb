                      # 线性表

                      # 定义

                      线性表是 $n\left(n>0\right)$ 个数据元素的有限序列。除第一个元素外都有一个直接前驱，除最后一个元素外都有一个直接后继。

                      $$L=\left(a_{1},a_{2},\cdots,a_{n}\right)$$

                      # 特点

                      • 元素个数有限。

                      • 逻辑上有顺序性。

                      • 元素都是数据元素。

                      • 数据类型都相同，占据相同的存储空间。

                      • 表中元素具有抽象性，只有逻辑关系。

                      # 基本操作

                      1. 初始化表。

                      2. 销毁表。

                      3. 线性表置空。

                      4. 线性表的长度。

                      5. 按位查找。

                      6. 插入操作。

                      7. 删除操作。

                      8. 按值查找。

                      9. 输出表。

                      public interface LinearTable {
    // 销毁线性表
    boolean remove();
    // 判断线性表是否为空
    boolean isEmpty();
    // 获取线性表的第i个元素
    int get(int i);
    // 向线性表中插入一个元素
    boolean insert(Object o);
    // 线性表的长度
    int size();
    // 打印线性表
    void print();
    // 按值查找
    int findValue(Object o);
    // 线性表置空
    void clear();
}

                      提示

                      无论是顺序表还是链式表，基本都要实现这几种操作。在 Java 中，我们选择使用接口来作为线性表的特征，后续顺序表和链表需要实现这些接口。

                      # 顺序表

                      使用连续的存储单元存储线性表中的元素。顺序表的逻辑结构和物理顺序相同。

                      顺序表在表中大部分会使用数组来进行实现，数组可以静态分配也可以动态分配，静态分配表的大小就固定死了，一旦空间占满就可能导致溢出问题。动态分配内存会在空间满后开辟新的内存空间，因此没有溢出问题。

                      顺序表的特点是随机访问，通过首地址和下标找到指定的元素，时间复杂度为 $O(1)$。插入和删除需要每一个元素都移位，时间复杂度为 $O(n)$。

                      时间复杂度

                      这边的时间复杂度一般是指平均时间复杂度。

                      # 链式表

                      链表是使用任意存储单元存储线性表的数据元素，除了需要存储本身信息，还需要存储后继节点的信息。

                      # 单链表

                      一般的线性表的链式存储可称为单列表，其中的保存的节点信息是后继节点的位置。

                      # 栈

                      栈是仅在表尾进行插入或删除的线性表，先进后出的线性表，表头端称为栈底，表尾端称为栈顶。

                      # 队列

                      队列是一种先进先出的线性表，允许在表头进行插入，表头进行删除。
