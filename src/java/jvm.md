## 简介

Java Virutal Machine 的简称，JVM 负责运行 Java 字节码文件。

### 功能

1. 解释和运行：将字节码文件翻译成机器码。
2. 内存管理：自动为对象、方法分配内存；垃圾自动回收。
3. 即时编译：对热点代码优化，提升执行效率。

## 组成部分

1. 类加载器
2. 运行时数据区
3. 执行引擎
4. 本地库接口

只要完成了 JVM 的规范，就可以称为 JVM。

### 常见JVM

1. HotSpot VM
2. Graal VM



### JVM组成

1. 类加载器（Class Loader）：用于将.class文件加载到内存中。
2. 运行时数据区（Runtime Data Area）：用于存储运行时的数据，包括方法区、堆、Java栈、PC寄存器、本地方法栈。
3. 执行引擎（Execution Engine）：负责执行类加载器加载的字节码，将字节码解释成机器码。
4. 本地库接口（Native Interface）：融合不同的编程语言为Java所用，它的初衷是融合C/C++程序。
5. 本地方法库（Native Method Libraries）：与本地库接口密切相关，执行引擎通过本地方法接口来调用本地方法库，它包含了一些其他语言写的代码。

## 垃圾回收


在 C/C++ 中，一个对象不适用需要手动释放，否则会出现内存泄漏，释放对象的过程为垃圾回收，程序员手动回收的方式为手动回收。长时间不回收可能导致内存溢出。

### 自动垃圾回收

Java 引入自动垃圾回收（ GC ）机制，垃圾回收器主要负责对堆上内存进行回收，Go、Python 都有垃圾回收器。

### 回收区域

方法区的类被卸载，需要所有实例对象都被回收，堆中不存在实例对象和子类对象、类加载器被回收、该类的Class对象没有被引用。

通过 `System.gc()` 方法来手动触发回收，

Java中对象被回收，是根据是否被引用决定的。

### 垃圾回收算法

#### 引用计数法

区分对象是否被引用时，将每个对象设置一个计数器，当对象被引用时 +1，取消引用时 -1。

引用计数器实现简单，但是对系统性能有影响，存在循环引用问题。

:::tip 循环引用
循环引用就是 A 引用 B ，B 同时引用 A会出现对象无法回收的问题。
:::

#### 可达性分析算法

Java 使用可达性分析算法判断对象是否可以回收，将对象分为根对象和普通对象，对象和对象之间存在引用关系。

GC Root 对象：
1. 线程 Thread 对象
2. 系统类加载器加载的 java.lang.Class 对象
3. 监视器对象
4. 全局对象

#### 评价标准

垃圾回收会停止所有用户线程，这个过程是 Stop The World。

1. 吞吐量：执行用户代码时间/（执行哦用户代码时间+ GC 时间）。
2. 最大暂停时间。
3. 堆使用效率。

#### 标记清除算法

1. 标记阶段：将存活对象进行标记。
2. 清除阶段：从内存中删除非存活对象。

优点是简单，缺点是碎片化问题，分配速度慢。

#### 复制算法

将空间分为 From 空间和 To 空间，每次在对象分配阶段，只能使用 From 空间。在垃圾回收 GC 阶段，将 From 中存活对象复制到 To 空间，将空间名互换。

只需遍历一次就将存活对象复制到 To 空间
